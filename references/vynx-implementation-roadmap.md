# VYNX Implementation Roadmap

Use this file when the user needs the standard phased VYNX rollout model.

## Diagnose

Objective:
- identify where high-value deals leak and why

Key work:
- deal audits
- stakeholder interviews
- stage leakage analysis
- distributor variance review
- applications engineering and service handoff assessment

Key outputs:
- diagnostic readout
- root-cause map
- baseline KPI pack
- opportunity-stage leakage view

Leadership checkpoint:
- agreement on the real failure points and priority segments or channels

## Design

Objective:
- define the operating model required to fix the leakage

Key work:
- stage-gate design
- decision-rule design
- artifact design
- distributor tiering model
- review cadence design
- CRM and CPQ requirement definition

Key outputs:
- stage architecture
- proposal-readiness rubric
- templates
- tiering model
- management cadence
- dashboard blueprint

Leadership checkpoint:
- approval of the future-state operating system and pilot scope

## Pilot

Objective:
- prove the system on live strategic deals in a controlled environment

Key work:
- select pilot region or distributors
- activate templates
- run reviews
- enforce proposal blocks
- track leading indicators

Key outputs:
- pilot dashboard
- deal-review logs
- distributor scorecards
- field feedback
- refined rules

Leadership checkpoint:
- evidence that pilot deals show better discipline and better progression

## Enforce

Objective:
- move from optional usage to operating standard

Key work:
- manager inspection
- CRM and CPQ stage controls
- tier consequences
- applications engineering and service engagement rules
- stalled-deal coding discipline

Key outputs:
- embedded workflow controls
- governance rhythm
- escalation paths
- compliance reporting

Leadership checkpoint:
- confirmation that the process is being used to make decisions, not just complete forms

## Scale

Objective:
- expand the system across regions, segments, distributors, and product lines

Key work:
- broaden rollout
- refine thresholds
- rebalance support by tier
- refresh training and audit loops
- standardize executive reporting

Key outputs:
- scaled operating playbook
- enterprise dashboard
- tier refresh cycle
- audit cadence

Leadership checkpoint:
- confirmation that the system is durable, repeatable, and delivering conversion improvement at scale
